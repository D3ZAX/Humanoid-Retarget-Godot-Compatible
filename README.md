Humanoid Retarget (Godot Compatible)Humanoid Retarget is a specialized Blender tool designed to simplify the process of remapping, aligning, and retargeting animations between different humanoid armatures. It is particularly optimized for Godot Engine's humanoid bone naming conventions but flexible enough for any humanoid rig.🚀 Key FeaturesSmart Auto-Detection: Uses advanced geometric analysis (directional chains, dot product validation, and vertex group counts) to automatically map your source and target bones.One-Click Pose Alignment: Instantly aligns a target armature's pose to match a source armature's orientation, solving the "T-Pose vs. A-Pose" discrepancy.Safe Rest Pose Application: Applies the current pose as the new Rest Pose while automatically adjusting associated meshes so they don't "explode" or distort.Edit-Mode Bone Roll Copy: Synchronizes local bone axes between armatures while preserving the world-space position of child objects.Symmetry Logic: Intelligently identifies Left/Right pairs and maps them based on naming differences (e.g., .L to .R).JSON Import/Export: Save and load your bone mapping configurations to reuse across different projects.🛠 InstallationDownload the blender_humanoid_retarget.py file.Open Blender, go to Edit > Preferences > Add-ons.Click Install... and select the .py file.Enable the add-on: Animation: Humanoid Retarget.Find the panel in the 3D View Sidebar (N-panel) under the Humanoid tab.📖 How to Use1. Bone MappingIn the Humanoid panel, select your Source Armature (the one with the animation) and Target Armature (the one you want to fix).Auto-Detect: Click Detect Source/Target by Hips. The plugin will analyze the bone hierarchy starting from the Hips to fill the mapping list.Manual Tweak: If any bone is missed, use the search box in the list to manually assign it.2. Aligning the PoseIf your target model is in a different pose (e.g., Source is T-Pose, Target is A-Pose):Click Align Target Pose (Godot).The plugin will calculate the rotational difference between the bone chains and snap the target bones into alignment with the source.3. Setting a New Rest PoseTo make the current alignment permanent:Select the Target Armature.Click Apply Rest Pose (Keep Mesh).What happens: The plugin applies the modifier on the mesh, sets the armature's rest pose, and then re-adds the armature modifier. Your model's default "zero" pose is now updated without visual distortion.4. Correcting Bone RollsTo ensure your local $X/Y/Z$ axes match the source (crucial for clean rotation math):Click Copy Bone Roll & Keep Children.This will align the Edit-Mode bones to match the source's orientation while ensuring any attached objects (like weapons or hats) stay in their correct world-space positions.📋 Bone Naming SupportThe plugin includes a built-in dictionary (GODOT_BONEMAP) that recognizes common naming conventions from:MixamoRigifyUnreal Engine (Mannequin)Unity (Mecanim)Standard Blender naming (.L / .R)⚠️ RequirementsBlender 4.5.0 or higher.Armatures should ideally be oriented with $+Y$ as the bone's primary axis (Blender Standard).📄 LicenseThis project is authored by D3ZAX.
